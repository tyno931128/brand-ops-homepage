# 2026-05-28 10:10 KST 최종 성장 브리프

기준 시각: 2026-05-28 10:10 KST

목표: 랜딩페이지와 홈페이지 제작을 사업화해 실제 수익으로 연결한다. 핵심 포지셔닝은 단순 제작자가 아니라 **브랜딩 문장, 신뢰 정보, 예약/문의 동선, SNS/검색 유입, Notion 운영판을 함께 정리하는 AI 브랜딩 홈페이지 제작자**다.

## 최종 판단

지금 홈페이지는 `무료 진단 -> 7일 브랜딩 랜딩 -> 30일 운영 개선`이라는 상품 흐름으로 정리되어 있다. 3D와 애니메이션은 유지하되, 고객이 실제로 맡기는 이유는 화려함보다 다음 4가지가 되어야 한다.

1. 내가 어떤 업종을 잘 도와주는지 바로 보인다.
2. 무료 진단을 받으면 무엇이 나오는지 구체적이다.
3. 홈페이지가 예약/문의/상담으로 이어지는 흐름을 설계한다.
4. 리서치와 Notion 정리, SNS 콘텐츠까지 운영형으로 이어진다.

## 진행한 리서치

### 시장/판매 방식

- 크몽, Upwork, Page.e, Webflow/Notion 기반 상품, 랜딩페이지 감사 서비스, 웹디자인 제안 템플릿을 조사했다.
- 시장은 이미 `빠른 홈페이지 제작`, `저가 랜딩페이지`, `노코드/Notion 홈페이지`로 붐빈다.
- 차별화는 `AI 리서치 팀 + Notion 지식관리 + 브랜딩 랜딩 + SNS/검색 운영`으로 잡는 것이 맞다.

### 우선 타겟

- 기능 요구가 복잡한 IT 기업보다 필라테스/PT, 피부관리/뷰티, 학원/클래스, 공방, 1인 전문가, 소규모 병원/상담센터가 초기 수익화에 더 적합하다.
- 이유는 홈페이지 기능보다 첫 문장, 가격/프로그램, 후기, FAQ, 예약 버튼, 위치/지역 신뢰 정보만 정리해도 전환 개선 여지가 크기 때문이다.
- 의료/상담 업종은 수익 가능성은 있지만 광고 표현 규제 리스크가 있어 과장, 치료효과 보장, 후기/전후 비교 표현을 피해야 한다.

### 랜딩/브랜딩 전환 원리

- 전환형 랜딩은 한 가지 오퍼와 한 가지 CTA가 강하다.
- 첫 화면에서 `누구를 위한 서비스인지`, `무엇을 받는지`, `다음 행동이 무엇인지`가 보여야 한다.
- 로컬 서비스는 모바일 속도, 실제 사진/후기/지역 신뢰, 전화/예약/상담 CTA가 핵심이다.
- 애니메이션과 3D는 첫인상에는 도움을 줄 수 있지만, 속도와 이해도를 해치면 전환에 불리하다.

### SNS/SEO/로컬 검색

- 필라테스/뷰티 업종은 Instagram이 분위기와 사례를 보여주는 앞단 채널이고, 랜딩페이지는 가격/프로그램/예약/FAQ를 정리하는 전환 허브가 되어야 한다.
- Google Business Profile, 네이버 스마트플레이스, 네이버 서치어드바이저, Google Search Console은 배포 후 기본 체크 항목이다.
- 상위 노출 보장은 금지하고, 검색 등록/기본 최적화/정보 일관성/측정 준비로 표현해야 한다.

## 홈페이지 반영 내용

현재 홈페이지는 다음 흐름으로 구성되어 있다.

1. Hero: 문의로 이어지는 브랜딩 홈페이지 제작, 무료 진단 CTA
2. Target: 바로 문의/예약으로 이어지는 업종 집중
3. Free Diagnosis: 무료 진단 산출물 3개
4. AI Engine: 리서치, Notion, 브랜딩, 웹사이트가 연결되는 3D 장면
5. Operating Screens: 리드 탐지, 자료 정리, 화면 제작, SNS 전환
6. Sample Audits: 실제 사례가 아닌 무료 진단 방식 샘플
7. User Lens: 고객이 외주를 맡기는 기준
8. Regulated Services: 병원/상담 업종 표현 안전장치
9. Packages: 무료 진단, 7일 브랜딩 랜딩, 30일 운영 개선
10. Contact: 업종, 현재 링크, 원하는 문의 방식 중심 무료 진단 신청

## 오늘 바로 실행할 우선순위 5개

1. 실제 문의 수신 경로 확정
   - `hello@example.com`을 실제 이메일, 카카오 채널, Tally/Google Form, 또는 Notion DB 연동 폼으로 교체한다.

2. 첫 타겟 하나만 고정
   - 추천 1순위는 `필라테스/PT/피부관리`다. 인스타 기반으로 리드 탐색이 쉽고, 가격/후기/예약 동선 정리의 가치가 바로 보인다.

3. 무료 진단 샘플 3개 작성
   - 실제 연락 전, 필라테스 1개, 피부관리 1개, 학원 1개를 가상 또는 공개 링크 기준 샘플로 만들어 홈페이지/Notion에 보여줄 수 있게 한다.

4. GitHub/Vercel 공개 준비
   - 민감 정보 점검 후 GitHub remote를 연결하고, Vercel project link를 만든다. production URL이 정해지면 metadata canonical/OG URL을 추가한다.

5. Notion Leads CRM 실제 DB 생성
   - `docs/notion-leads-crm-schema.md` 기준으로 Leads, Audit Reports, Outreach Log를 만든다. 이후 무료 진단 신청이 들어오면 자동/수동으로 기록한다.

## 남은 리스크

- 문의 경로가 placeholder라 지금 배포하면 실제 리드가 유실된다.
- 샘플 진단은 아직 실제 고객 성과가 아니므로 과장하면 신뢰를 잃는다.
- 의료/상담 업종은 표현 규제 리스크가 있어 별도 문구 검토가 필요하다.
- 3D/애니메이션은 멋져 보이지만 모바일 성능을 계속 확인해야 한다.
- GitHub/Vercel 공개 전 Notion 링크, 고객 정보, API key, 비공개 DB ID가 섞이지 않았는지 점검해야 한다.
- 가격은 아직 가설이다. 첫 3건의 상담 반응을 보고 `무료 진단 -> 7일 제작` 전환 가격을 조정해야 한다.
- SEO/검색 등록은 결과 보장이 아니라 기본 세팅과 정보 일관성으로만 설명해야 한다.
