# Design System and Motion Guide

기준일: 2026-05-28

이 문서는 개인 브랜딩 기반 랜딩페이지/홈페이지 제작 사업의 시각 방향을 정의한다. 목표는 화려한 3D 포트폴리오가 아니라, 방문자가 **이 사람에게 맡기면 내 사업의 신뢰와 문의 흐름이 좋아지겠다**고 느끼게 만드는 것이다.

## 디자인 원칙

1. 결과 먼저
   첫 화면은 홈페이지 제작 자체가 아니라 `문의가 생기는 브랜딩 랜딩페이지`를 말한다.

2. 설명보다 시연
   긴 소개글보다 업종 입력, Before/After, 화면 변환, 문의 흐름 같은 움직이는 데모를 먼저 보여준다.

3. 3D는 기술력의 증거로만
   장식용 3D는 줄이고, `리서치 -> 브랜딩 -> 랜딩 -> 문의 전환`이 작동하는 시스템처럼 보일 때만 쓴다.

4. 고급스럽지만 차분하게
   과한 네온, 과한 회전, 게임 같은 효과는 피한다. 밝은 배경, 선명한 타이포, 제한된 포인트 컬러를 쓴다.

5. 타겟을 좁혀 말하기
   모든 홈페이지 제작이 아니라 1인 사업자, 소규모 병원, 학원, 스튜디오, 로컬 브랜드처럼 홈페이지가 매출에 바로 연결되는 대상을 우선한다.

6. 신뢰는 운영 방식에서 나온다
   “잘 만들어요”보다 리서치하고, 문구를 고치고, SEO와 SNS까지 반복 개선한다는 프로세스를 디자인으로 보여준다.

## 3D와 애니메이션 사용 기준

| 영역           | 사용 강도 | 기준                                                        |
| -------------- | --------- | ----------------------------------------------------------- |
| Hero           | 강함      | 브랜드 진단, 랜딩 구조, 문의 발생 흐름이 한눈에 보여야 한다 |
| 업종 입력 데모 | 강함      | 업종 칩 선택에 따라 문제/문구/개선안이 바뀐다               |
| 서비스 설명    | 약함      | 카드 hover, 단계 reveal 정도로 제한한다                     |
| Before/After   | 중간      | 오래된 화면이 개선 화면으로 바뀌는 장면을 보여준다          |
| 패키지/가격    | 낮음      | 비교와 신뢰가 중요하므로 정적으로 유지한다                  |
| 문의 CTA       | 낮음      | 버튼, 입력창, 응답 예시 정도만 쓴다                         |

## 섹션별 인터랙션

| 섹션           | 목적                                | 인터랙션                                         |
| -------------- | ----------------------------------- | ------------------------------------------------ |
| Hero           | 5초 안에 무엇을 파는지 이해         | 3D 패널이 리서치, 브랜딩, 랜딩, 문의 순서로 점등 |
| 업종 입력 데모 | 내 사업에 적용된다는 느낌           | 업종 칩 선택 -> 문구/문제/개선안 변경            |
| 타겟 섹션      | 누구를 위한 서비스인지 명확화       | 병원/학원/1인기업/스튜디오 카드 전환             |
| 문제 제기      | 기존 홈페이지의 손실 인식           | 오래된 홈페이지 화면이 개선 화면으로 전환        |
| 제작 방식      | 단순 제작자가 아닌 운영 파트너 인식 | 5단계 프로세스가 스크롤에 맞춰 순차 활성화       |
| AI 운영 시스템 | 차별점 전달                         | 리서치 AI, 문서 AI, 마케팅 AI가 연결되는 3D 노드 |
| 결과물 예시    | 맡겼을 때 무엇을 받는지 확인        | 랜딩/SEO/SNS/Notion 운영판 미리보기              |
| 패키지         | 구매 판단                           | 정적이고 명확한 가격/범위 비교                   |
| 문의           | 행동 유도                           | 짧은 폼 + 내 업종 진단 받기 CTA                  |

## Figma 협업 산출물

Figma 파일을 만들 때는 다음 페이지 구조로 시작한다.

1. Design Tokens
   색상, 타이포, 간격, radius, motion token.

2. Landing Wireframes
   desktop/mobile 섹션 구조.

3. Component Library
   Header, CTA, Demo Input, Proof Card, Package Card, Contact Form.

4. Motion Storyboard
   Hero 3D, 업종 입력 데모, Before/After 전환.

5. Copy Matrix
   타겟별 헤드라인, 문제 문장, CTA 문구.

6. QA Board
   디자인, 반응형, 접근성, 성능 체크리스트.

## 디자인 QA 체크리스트

- 첫 화면에서 무엇을 해주는 사람인지 5초 안에 보이는가
- H1이 서비스 결과를 말하는가
- CTA가 “문의하기”보다 구체적인 행동을 유도하는가
- 글보다 화면 시연이 더 먼저 보이는가
- 3D가 의미 없이 떠다니지 않는가
- 모바일에서 텍스트가 버튼/카드 밖으로 넘치지 않는가
- 카드 radius가 과하게 둥글지 않은가
- 색상이 한 가지 계열로만 보이지 않는가
- 애니메이션이 읽기를 방해하지 않는가
- `prefers-reduced-motion` 대응이 가능한가
- 패키지 영역은 충분히 정적이고 비교하기 쉬운가
- 문의 폼은 3개 이하 입력으로 시작 가능한가
- SEO 제목/설명이 타겟 키워드를 포함하는가
- 실제 외주 고객이 “내 사업도 이렇게 바뀌겠다”고 느끼는가
- 디자인이 포트폴리오가 아니라 판매 페이지로 작동하는가
