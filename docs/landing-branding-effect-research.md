# Landing Page and Branding Effect Research

기준일: 2026-05-27

## 목적

이 문서는 좋은 랜딩페이지가 어떤 심리적 효과를 만들고, 브랜딩이 어떤 이미지를 만드는지 계속 조사하기 위한 기준 문서다. 목표는 예쁜 페이지를 모방하는 것이 아니라 **방문자가 더 빨리 이해하고, 더 쉽게 믿고, 더 자연스럽게 문의하게 만드는 구조**를 찾는 것이다.

## 계속 볼 질문

1. 첫 화면에서 무엇을 한 문장으로 말하는가?
2. 방문자가 5초 안에 “나에게 필요한 서비스”라고 느끼는가?
3. 제품/서비스가 실제로 작동하는 장면이 보이는가?
4. 애니메이션과 3D가 의미를 설명하는가, 아니면 장식으로만 보이는가?
5. 신뢰는 어디서 생기는가? 고객사, 사례, 수치, 후기, 가격, 과정, 보증, 투명성 중 무엇인가?
6. CTA는 하나의 행동으로 집중되는가?
7. 모바일에서 첫 화면과 CTA가 빠르게 이해되는가?
8. 페이지 속도와 요소 수가 전환을 방해하지 않는가?

## 현재 관찰: Channel.io

관찰:

- 첫 화면에서 `한국 No.1 고객 상담 솔루션`이라는 큰 카테고리 문장을 먼저 보여준다.
- 오른쪽에는 추상 이미지가 아니라 제품 데모처럼 보이는 입력형 화면을 둔다.
- 아래에는 고객사 로고를 바로 배치해 “이미 쓰는 곳이 많다”는 신뢰를 만든다.
- 다음 섹션은 긴 기능 설명이 아니라 `100건의 문의, 상담원이 응대할 건 20건`처럼 결과 중심 문장으로 이어진다.

심리적 효과:

- 사용자는 “무슨 서비스인지”를 빠르게 이해한다.
- 데모 화면이 있어서 기능을 상상하기 쉽다.
- 로고와 수치가 있어 과장된 주장처럼 느껴지지 않는다.
- 텍스트보다 장면과 결과 문장이 기억에 남는다.

우리 적용:

- 첫 화면 문장을 `홈페이지, 만들고 끝내지 않습니다.`로 줄였다.
- 오른쪽에 업종을 입력해보는 브랜드 진단 데모를 배치했다.
- 긴 설명 카드보다 `What I Do`, `AI Engine`, `Operating Screens`, `Before/After` 순서로 장면 중심 구조를 만들었다.
- 실제 고객 사례가 생기면 타겟 칩 대신 로고/사례/수치로 교체한다.

## 랜딩페이지 효과 기준

### 1. 이해 속도

좋은 랜딩페이지는 방문자가 긴 설명을 읽기 전에 다음을 알아야 한다.

- 누구를 위한 서비스인가?
- 어떤 문제를 해결하는가?
- 결과가 무엇인가?
- 다음 행동은 무엇인가?

현재 우리 페이지의 기준 문장:

> 홈페이지, 만들고 끝내지 않습니다.

이 문장은 홈페이지 제작자들과 다르게 “계속 개선한다”는 차이를 먼저 만든다.

### 2. 신뢰

Baymard의 제품 페이지 UX 연구는 구매 결정의 중심 페이지에서 구조적 UX 문제가 전환 실패로 이어질 수 있음을 보여준다. 특히 구매 영역, 리뷰, 배송/반품, 명세 정보처럼 사용자가 불안을 해소하는 정보가 약하면 문제가 커진다.

우리 서비스에 번역하면 신뢰 요소는 다음과 같다.

- 실제 작업 과정
- 가격 범위
- 수정 범위
- Before/After
- 무료 진단 예시
- Notion 운영판 미리보기
- 고객 질문과 답변

### 3. 속도와 밀도

Google의 모바일 랜딩페이지 연구는 모바일 방문자가 느린 페이지에서 빠르게 이탈하고, 페이지 요소가 많아질수록 전환 가능성이 떨어질 수 있음을 보여준다.

우리 적용 기준:

- 첫 화면에 너무 많은 카드와 긴 설명을 넣지 않는다.
- 3D와 애니메이션은 한 번에 너무 많이 쓰지 않는다.
- 모바일에서는 긴 문단보다 짧은 결과 문장과 화면 이미지를 우선한다.

### 4. CTA 집중

HubSpot의 CRO 가이드는 CTA가 눈에 잘 띄고 행동 중심이어야 하며, 랜딩페이지는 하나의 전환 행동에 집중해야 한다고 설명한다.

우리 적용 기준:

- 기본 CTA는 `무료 홈페이지 진단 받기` 하나로 둔다.
- 보조 CTA는 `운영 화면 보기`처럼 이해를 돕는 행동으로 둔다.
- 패키지마다 별도 구매 버튼을 남발하지 않는다.

## 브랜딩이 만드는 이미지

| 이미지 | 만드는 요소                            | 우리 페이지 적용                        |
| ------ | -------------------------------------- | --------------------------------------- |
| 전문성 | 간결한 핵심 문장, 가격 범위, 과정 공개 | 첫 문장, 패키지, What I Do              |
| 기술력 | 3D 엔진, 입력형 데모, 운영 패널        | BrandDemo, HeroScene, Operating Screens |
| 안정감 | 밝은 배경, 충분한 여백, 명확한 CTA     | 전체 밝은 톤, 둥근 제품형 UI            |
| 실행력 | Before/After, 리서치-제작-개선 루프    | Before/After, AI Engine                 |
| 친근함 | 업종 입력, 샘플 칩, 쉬운 표현          | 입력형 데모, 타겟 칩                    |

## 애니메이션/3D 사용 기준

사용해도 좋은 경우:

- 작동 방식을 설명한다.
- 데이터가 이동하거나 정리되는 흐름을 보여준다.
- 제품 화면처럼 보인다.
- 사용자의 시선이 CTA로 이동하게 돕는다.

줄여야 하는 경우:

- 글보다 움직임이 더 시끄럽다.
- 페이지가 느려진다.
- 모바일에서 핵심 CTA를 밀어낸다.
- “멋있다”는 느낌만 있고 무엇을 하는지 설명하지 못한다.

- Channel.io: 첫 화면 데모, 고객사 로고, 결과 중심 메시지
- Framer: 인터랙션과 빠른 제작 이미지
- Webflow: 전문적 제작/소유권/디자인 시스템 이미지
- Linear: 미니멀한 고신뢰 SaaS 브랜딩
- Toss: 쉬운 언어와 명확한 행동 유도
- 국내 프리랜서/에이전시 랜딩페이지: 가격, 과정, 사례 제시 방식
- 병원/학원/필라테스 상위 노출 홈페이지: 로컬 SEO와 CTA 구조

## 이번 주 실험

1. 첫 화면 CTA를 `무료 홈페이지 진단 받기`로 유지하고 클릭/상담 반응을 본다.
2. 무료 진단 결과 샘플 섹션을 추가한다.
3. Before/After를 실제 업종별 예시 화면으로 바꾼다.
4. 타겟 칩 대신 실제 사례/로고가 생기면 즉시 교체한다.
