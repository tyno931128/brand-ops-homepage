# Homepage Structure

이 문서는 현재 홈페이지의 메시지 흐름과 코드 구조를 설명한다.

## 핵심 메시지

> 홈페이지, 만들고 끝내지 않습니다. 고객이 이해하고, 신뢰하고, 문의할 때까지 첫 문장부터 화면, SEO, SNS 콘텐츠까지 계속 다듬습니다.

이전 버전은 설명 카드가 많아 무엇을 말하려는지 흐려졌다. 현재 버전은 랜딩페이지답게 `한 문장`, `한 행동`, `한 개의 큰 제품 데모`를 첫 화면에 배치한다.

## 참고한 랜딩 원칙

- 첫 화면은 한 문장과 한 행동만 강하게 둔다.
- 제품이나 운영 화면이 글보다 먼저 보여야 한다.
- 긴 설명보다 움직이는 데모, 로고/타겟, 결과 수치, Before/After가 이해를 빠르게 만든다.
- 고객은 기능 목록보다 “내 문제가 이렇게 정리되는구나”를 보고 맡긴다.

## Route Structure

| Route | File               | Purpose              |
| ----- | ------------------ | -------------------- |
| `/`   | `src/app/page.tsx` | 공개 브랜딩 홈페이지 |

공통 레이아웃:

| File                  | Purpose                                     |
| --------------------- | ------------------------------------------- |
| `src/app/layout.tsx`  | HTML lang, Geist font, metadata, Open Graph |
| `src/app/globals.css` | Tailwind import, animation, 3D screen CSS   |

주요 컴포넌트:

| File                            | Purpose                           |
| ------------------------------- | --------------------------------- |
| `src/components/brand-demo.tsx` | 첫 화면의 입력형 브랜드 진단 데모 |
| `src/components/hero-scene.tsx` | Three.js 기반 AI 운영 엔진 시각화 |

## Page Sections

### 1. Header

목적:

- Brand Ops라는 제품/스튜디오 이름을 노출한다.
- `작동 방식`, `운영 화면`, `패키지`, `무료 진단`으로 이동한다.

### 2. Hero

목적:

- 방문자가 5초 안에 “홈페이지를 만들고 끝내는 사람이 아니라, 문의가 생길 때까지 개선하는 사람”이라고 이해하게 한다.
- 오른쪽에 입력형 데모를 배치해 사용자가 자신의 업종을 넣어보게 한다.

핵심 문장:

- 홈페이지, 만들고 끝내지 않습니다.

CTA:

- 무료 홈페이지 진단 받기
- 운영 화면 보기

### 3. Target Marquee

목적:

- 어떤 업종을 대상으로 하는지 빠르게 보여준다.
- 고객 로고가 없는 초기 단계이므로, 실제 타겟 업종을 움직이는 칩 형태로 대체한다.

타겟:

- 1인 전문가
- 학원·클래스
- 필라테스·PT
- 뷰티샵
- 로컬 서비스
- 소규모 병원
- 공방
- 스튜디오

### 4. What I Do

목적:

- 제공 가치를 세 문장으로 압축한다.

흐름:

1. 무엇을 말해야 할지 정리
2. 보여지는 화면으로 제작
3. 문의가 생길 때까지 개선

### 5. AI Engine

목적:

- 뒤에서 AI 팀이 계속 리서치하고 정리하고 개선한다는 구조를 3D로 보여준다.
- HeroScene 캔버스를 제품 화면처럼 사용한다.

### 6. Operating Screens

목적:

- 설명보다 먼저 작동하는 화면을 보여준다.
- CSS 3D 카드와 스캔/데이터 라인 애니메이션으로 운영 화면의 느낌을 만든다.

화면:

- Research Radar
- Notion Command
- Landing Builder
- Content Engine

### 7. Before / After

목적:

- 고객이 맡기고 싶어지는 이유를 문제/개선 구조로 보여준다.
- 긴 설명 대신 “흩어져 있음 → 정리됨”의 대비를 만든다.

### 8. User Lens

목적:

- 실제 후기가 생기기 전까지 고객이 외주를 맡길 때 보는 판단 기준을 명확히 보여준다.
- 가짜 리뷰처럼 보이지 않게 `고객 판단 기준`으로 표현한다.

카드:

- 내 사업을 먼저 이해하는가
- 예쁜 화면보다 결과를 말하는가
- 맡긴 뒤에도 통제 가능한가
- 과하게 만들지 않는가

### 9. Packages

목적:

- 제작비가 아니라 성장 단계로 제안한다.

패키지:

- Starter
- Core
- Growth

### 10. Contact

목적:

- 무료 진단 요청으로 전환한다.
- 현재는 `mailto:` CTA이며 이후 Notion Leads DB 또는 별도 form action으로 대체한다.

## Visual Direction

현재 시각 방향:

- 밝은 제품형 랜딩페이지
- 큰 첫 문장
- 입력형 데모 UI
- 부드러운 색상 배경과 둥근 디바이스 목업
- 중간 구간은 어두운 3D AI 엔진으로 대비
- CSS 3D 패널과 스캔라인 애니메이션

Guardrails:

- 설명 카드가 너무 많아지면 바로 줄인다.
- 한 섹션에는 하나의 메시지만 둔다.
- 실제 고객 사례가 생기면 타겟 칩보다 사례/로고를 우선한다.
- 3D는 장식이 아니라 운영 흐름을 보여줄 때만 사용한다.
